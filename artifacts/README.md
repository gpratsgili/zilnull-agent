# artifacts

this is the shared durable output surface.

artifacts are not the same as memories.

- memories are what a spirit should know
- artifacts are what a spirit made, collected, captured, published, or filed

the live system tends to grow a few stable artifact subtrees:

- `archive/` for non-destructive retirement of old work
- `dreamtime/` and `nightly/` for ritual output notes and overnight consolidation residue
- `essays/`, `notes/`, `reports/`, and `research/` for durable written work
- `library/` for collected source material, captures, and PDFs
- `media/` for attachment-ready graphics and rendered visual guides
- `network/` for inscribed static sites and their attached local threads
- `tests/` for smoke artifacts and validation outputs that should persist outside the current run

project-specific folders can exist too. these generic roots are just the common skeleton.
